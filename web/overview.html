Project Overview

This project combines a family-systems framework, a normalized master dataset, and a repeatable multi-platform AI analysis process into one unified investigation dashboard.

Family First Cross-Platform AI Structured Dataset Repeatable Analysis Child-Centered

Overarching High-Level Vision

The high-level vision of the project is to create a disciplined, transparent, and comparable way to review family communication patterns without relying on one person's interpretation alone.

Instead of informal impressions, the project turns message history into structured data, applies the same questions to multiple AI platforms, and compares outputs side by side. The intended outcome is a clearer view of communication roles, influence, conflict patterns, risks to child stability, and practical recommendations for improving the system.

At its core, the project is built to keep the children as the priority, preserve role clarity, reduce triangulation, and create a repeatable decision-support framework that is more neutral than isolated opinions or fragmented message threads.

How the Project Fits Together

1. Cargill Table Framework

Defines the Family Table Priority Model and explains who sits at the table, who supports the table, and who should remain outside core family decision-making.

  • Tier 1: Primary family
  • Tier 2: Close kin
  • Tier 3: External adults

2. Master Dataset Design

Defines how raw message exports are normalized into participants, conversations, messages, incidents, JSON packets, prompts, and comparable AI outputs.

  • Raw exports
  • Normalized CSV tables
  • Shared JSON packet
  • Standard prompts and scoring

Open Master Dataset page

3. AI Comparison Workflow

Uses the same dataset, same prompt, same scoring system, and same output format for ChatGPT, Grok, and Gemini so findings can be compared fairly.

  • Comparable inputs
  • Comparable questions
  • Comparable outputs

4. Dashboard + Automation

Brings the framework and dataset into one interface with upload tools, analysis runners, scoring views, timelines, and exportable reports.

  • Investigation dashboard
  • Timeline and risk scoring
  • Exportable reporting

Why This Approach Matters

  • It reduces reliance on one-sided interpretation.
  • It makes communication analysis more structured and reviewable.
  • It ties every conclusion back to shared data and shared prompts.
  • It keeps the project focused on family systems and the best interest of the children.
  • It creates a reusable model that can be updated as more communication data is added.
